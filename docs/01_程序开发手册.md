## 1. 系统架构概述

### 1.1 系统组件

| 组件 | 技术栈 | 说明 |
|------|--------|------|
| **bend-platform** | Spring Boot + MySQL | 后端管理平台 |
| **bend-platform-web** | Vue 3 + Element Plus | 前端管理界面 |
| **bend-agent** | Python + OpenCV | 客户端自动化程序 |
| **MySQL** | 8.0+ | 数据库 |
| **WebSocket** | - | Agent与平台实时通信 |

## 3. 后端服务部署

### 3.1 数据库安装与配置

#### Ubuntu / Debian

```bash
# 安装MySQL
sudo apt update
sudo apt install mysql-server

# 启动MySQL
sudo systemctl start mysql
sudo systemctl enable mysql

# 安全配置
sudo mysql_secure_installation

# 登录MySQL
sudo mysql -u root -p
```

#### Windows

1. 下载 [MySQL Installer](https://dev.mysql.com/downloads/installer/)
2. 选择 MySQL Server 8.0
3. 设置root密码
4. 完成安装

### 3.2 创建数据库

```sql
-- 登录MySQL后执行
CREATE DATABASE bend_platform DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'bend_user'@'%' IDENTIFIED BY 'YourStrongPassword123!';
GRANT ALL PRIVILEGES ON bend_platform.* TO 'bend_user'@'%';
FLUSH PRIVILEGES;
USE bend_platform;
```

### 3.3 后端配置文件

编辑 `bend-platform/src/main/resources/application.yml`:

```yaml
server:
  port: 8090

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/bend_platform?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: bend_user
    password: YourStrongPassword123!
    driver-class-name: com.mysql.cj.jdbc.Driver

  servlet:
    multipart:
      max-file-size: 100MB
      max-request-size: 100MB
```

### 3.4 构建后端

```bash
cd d:\auto-xbox\team-management\bend-platform

# 方式一：Maven构建
mvn clean package -DskipTests

# 方式二：IDE构建
# 在IDE中右键 pom.xml -> Run Maven -> package
```

构建成功后，jar包位于 `target/bend-platform-1.0.0.jar`

### 3.5 运行后端服务

```bash
# 进入项目目录
cd d:\auto-xbox\team-management\bend-platform

# 直接运行
java -jar target/bend-platform-1.0.0.jar

# 或使用Spring Boot Maven插件
mvn spring-boot:run
```

### 3.6 验证后端服务

```bash
# 检查端口是否监听
netstat -an | findstr 8090

# 测试API
curl http://localhost:8090/api/health
```

### 3.7 生产环境部署（Linux）

```bash
# 创建服务用户
sudo useradd -m -s /bin/false bend

# 创建目录
sudo mkdir -p /opt/bend-platform
sudo chown bend:bend /opt/bend-platform

# 复制jar包
sudo cp target/bend-platform-1.0.0.jar /opt/bend-platform/

# 创建systemd服务文件
sudo nano /etc/systemd/system/bend-platform.service
```

内容：
```ini
[Unit]
Description=Bend Platform Service
After=network.target mysql.service

[Service]
Type=simple
User=bend
WorkingDirectory=/opt/bend-platform
ExecStart=/usr/bin/java -jar /opt/bend-platform/bend-platform-1.0.0.jar
Restart=always

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl start bend-platform
sudo systemctl enable bend-platform
```


访问 http://localhost:3090

### 4.3 生产环境构建

```bash
# 构建生产版本
npm run build

# 构建产物位于 dist/ 目录
```

### 4.4 Nginx 配置（生产环境）

```nginx
server {
    listen 80;
    server_name your-domain.com;

    root /var/www/bend-platform-web/dist;
    index index.html;

    # 前端路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API代理
    location /api/ {
        proxy_pass http://127.0.0.1:8090/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # WebSocket代理（Agent通信）
    location /ws/ {
        proxy_pass http://127.0.0.1:8090/ws/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

## 5. Agent打包与分发

### 5.1 Agent源码结构

```
bend-agent/
├── configs/
│   └── agent.yaml          # 配置文件
├── src/
│   ├── main.py            # 入口
│   └── agent/             # 核心代码
├── templates/             # 模板图像
├── requirements.txt       # 依赖
└── scripts/
    └── build.bat          # 打包脚本
```

### 5.2 安装打包工具

```bash
cd d:\auto-xbox\team-management\bend-agent
pip install pyarmor pyinstaller
```

### 5.3 执行打包

```bash
# 方式一：使用打包脚本
scripts\build.bat

# 方式二：手动执行
# 1. 加密代码
pyarmor gen --output dist/agent --assert all --assert call src/

# 2. 复制配置文件
copy /Y configs\* dist\agent\

# 3. 复制模板
xcopy /S /Q templates\* dist\agent\templates\

# 4. PyInstaller打包
pyinstaller --name BendAgent ^
    --add-data "dist\agent;agent" ^
    --add-data "configs;configs" ^
    --add-data "templates;templates" ^
    --hidden-import=aiohttp ^
    --hidden-import=websockets ^
    --hidden-import=cv2 ^
    --hidden-import=numpy ^
    --hidden-import=PIL ^
    --hidden-import=pyautogui ^
    --hidden-import=win32gui ^
    --hidden-import=win32ui ^
    --hidden-import=win32con ^
    --hidden-import=yaml ^
    --onefile ^
    --console ^
    dist\agent\main.py
```

### 5.4 打包产物

`dist\release\` 目录内容：

```
BendAgent/
├── BendAgent.exe      # 主程序
├── agent.yaml         # 配置文件
├── templates/         # 模板目录
└── README.txt        # 使用说明
```

### 5.5 商户分发流程

1. 将 `dist\release\` 打包成 `BendAgent_v{x.x.x}.zip`
2. 发给商户
3. 商户解压到任意目录
4. 修改 `agent.yaml` 配置
5. 运行 `BendAgent.exe`

## 7. 常见问题排查

### 7.1 后端启动失败

| 错误 | 解决方案 |
|------|---------|
| 端口被占用 | `netstat -ano \| findstr 8090` 查找进程并关闭 |
| 数据库连接失败 | 检查MySQL服务和配置 |
| 内存不足 | 增加JVM堆内存 `-Xmx512m` |

### 7.2 前端无法访问后端

| 检查项 | 命令 |
|--------|------|
| 后端是否运行 | `netstat -an \| findstr 8090` |
| API是否响应 | `curl http://localhost:8090/api/health` |
| 跨域配置 | 检查后端 CORS 配置 |

### 7.3 Agent无法连接

| 检查项 | 解决方案 |
|--------|---------|
| 网络连通性 | `ping 服务器IP` |
| 端口开放 | `telnet 服务器IP 8090` |
| WebSocket | 检查 `/ws/agent` 端点 |
| 防火墙 | 开放 8090, 8888 端口 |

### 7.4 数据库迁移

```bash
# 查看所有迁移脚本
ls db/

# 按顺序执行迁移
mysql -u root -p bend_platform < db/vip_config_deprecation_migration.sql
mysql -u root -p bend_platform < db/optimization_migration.sql
```
