### 1.1 三层架构设计

```
┌─────────────────────────────────────────────────────────────────┐
│                      bend-platform-web (前端)                     │
│  Vue 3 + Element Plus + Pinia + Vue Router                      │
│  ├── 页面视图 (Views)                                            │
│  ├── API 调用层                                                  │
│  ├── 状态管理 (Pinia Stores)                                     │
│  └── 路由 + 权限控制                                             │
└─────────────────────────────────────────────────────────────────┘
                              │
                              │ HTTP REST API / WebSocket
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                      bend-platform (后端)                         │
│  Spring Boot + MyBatis-Plus + JWT + WebSocket                    │

│  └── MySQL 数据库                                               │

                              │

                              ▼
┌─────────────────────────────────────────────────────────────────┐
```

## 二、功能模块分析

### 2.1 认证与授权

#### 前端认证流程
```javascript
// auth.js - 登录请求
authApi.login({ loginKey, password })
  → 后端返回 { token, userId, username, merchantId, role }

// auth.js - Token 存储
localStorage.setItem('token', token)
localStorage.setItem('user', JSON.stringify(user))
```

#### 后端认证流程
```java
// JwtAuthInterceptor.java - Token 验证
1. 从 Header 提取 "Bearer {token}"
2. jwtUtil.validateToken(token)
3. 解析 claims 获取用户信息
4. UserContext.setUserInfo(userInfo)
```

#### 角色权限
| 角色 | 前端标识 | 说明 |
|------|----------|------|
| platform_admin | isPlatformAdmin | 平台管理员 |
| owner | isOwner | 商户所有者 |
| admin | isAdmin | 商户管理员 |
| operator | isOperator | 操作员 |

**问题发现**：
- ⚠️ 角色字段存储为字符串，没有枚举约束
- ⚠️ 前端权限判断分散在各个视图

### 2.3 WebSocket 实时通信

#### Agent 连接流程
```
Agent 启动
    ↓
WSClient.connect(ws://host:port/{agentId}?agentSecret={secret})
    ↓
AgentWebSocketEndpoint.onOpen()
    ↓
validateAgent(agentId, agentSecret)
    ↓
AGENT_SESSIONS.put(agentId, session)
    ↓
sendMessage("connected", {...})
    ↓
broadcastToAdmins("agent_online", {...})
```

#### 心跳机制
- **Agent → 后端**：每 30 秒发送 heartbeat
- **后端 → Agent**：回复 heartbeat_ack
- **断开检测**：session.setMaxIdleTimeout(120000)

#### 问题发现：
- ✅ 心跳机制完整
- ⚠️ 后端没有定时检查 Agent 心跳超时的逻辑
- ⚠️ Agent 重连后，前端 WebSocket 推送依赖广播通知

### 2.4 Xbox 控制

#### SmartGlass 协议架构
```python
# XboxSession - 单个 TCP 连接
XboxStreamController.connect(ip, port)
    ↓
SmartGlass 协议握手
    ↓
维持长连接执行命令
```

#### 会话管理
```python
# XboxSessionManager - 连接池
_max_sessions = 100  # 最大并发
_sessions: Dict[str, XboxSession]  # IP → Session
_session_semaphore  # 信号量控制
```

#### 问题发现：
- ⚠️ 一个 IP 只能有一个会话，无法多开
- ⚠️ 没有视频流传输机制（只有命令控制）
- ⚠️ 前端无实时画面查看功能

## 六、稳定性分析

### 6.1 Agent 稳定性

| 机制 | 状态 | 说明 |
|------|------|------|
| 自动重连 | ✅ | 指数退避策略，最多 10 次 |
| 心跳保活 | ✅ | 30 秒间隔 |
| 优雅关闭 | ✅ | atexit + signal handler |
| 任务取消 | ✅ | asyncio.Event |

### 6.2 后端稳定性

| 机制 | 状态 | 说明 |
|------|------|------|
| 数据库连接池 | ❓ | 需要确认配置 |
| WebSocket 会话管理 | ✅ | ConcurrentHashMap |
| 异常处理 | ✅ | 全局异常拦截 |

### 6.3 问题发现

- ⚠️ **Agent 断线重连** - 前端无感知，需要刷新
- ⚠️ **任务超时** - 无任务超时机制
- ⚠️ **数据库事务** - 部分方法有 @Transactional
- ⚠️ **日志规范** - 无统一日志格式

| # | 问题 | 影响 | 修复方案 |

| 3 | 无操作日志 | 审计 | 记录关键操作 |
| 4 | Agent 心跳超时未处理 | 连接管理 | 后端定时检查 |
