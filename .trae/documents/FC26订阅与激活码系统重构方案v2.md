## 一、业务需求总结

### 1.1 激活码类型（5种）

| 类型 | 名称 | 绑定要求 | 扣点方式 |
|------|------|---------|---------|
| 1 | 流媒体账号包月 | **强制绑定**流媒体账号（多选） | 不扣点，按月计 |
| 2 | 游戏账号包月 | **强制绑定**游戏账号（多选） | 不扣点，按月计 |
| 3 | Xbox主机包月 | **强制绑定**Xbox主机（多选） | 不扣点，按月计 |
| 4 | 全功能包月 | **不限制**使用范围 | 不扣点，按月计 |
| 5 | 充值点数 | **不绑定**任何资源 | 按游戏账号数量扣点 |

### 1.2 激活码激活规则

1. **时间顺延规则**：
   - 多个激活码时间**不能重合**
   - 按激活顺序顺延（每月按30天计算）
   - 第N个激活码的开始时间 = 第(N-1)个激活码的到期时间 + 1天
   - 第N个激活码的到期时间 = 第N个激活码的开始时间 + 30天

2. **失效规则**：
   - 当前激活码到期后，如果没有其他有效激活码，则不能使用自动化功能

### 1.3 启动自动化校验规则

| 激活码类型 | 流媒体账号校验 | 游戏账号来源 | Xbox主机来源 |
|-----------|--------------|------------|-------------|
| 流媒体账号包月 | ✅ 必须是被绑定的流媒体账号 | 绑定流媒体账号下的所有游戏账号（默认全选） | 商户下所有在线Xbox主机（可选） |
| 游戏账号包月 | 不校验 | 必须是订阅绑定的游戏账号（过滤掉未绑定流媒体账号的） | 商户下所有在线Xbox主机（可选） |
| Xbox主机包月 | 不校验 | 绑定流媒体账号下的所有游戏账号 | 必须是订阅绑定的Xbox主机（必选） |
| 全功能包月 | 不校验 | 绑定流媒体账号下的所有游戏账号 | 商户下所有在线Xbox主机（可选） |
| 充值点数 | 不校验 | 绑定流媒体账号下的所有游戏账号 | 商户下所有在线Xbox主机（可选） |

### 1.4 VIP等级与价格体系

#### 1.4.1 VIP等级计算规则

```
VIP等级 = 根据商户"累计消费金额"计算（非累计充值）
```

#### 1.4.2 激活码生成时的价格体系

在**VIP分组（merchant_group）**中增加价格配置：

| 字段 | 说明 |
|------|------|
| vip_level | VIP等级（0, 1, 2, 3...） |
| window_original_price | 流媒体账号包月原价 |
| window_discount_price | 流媒体账号包月折扣价 |
| account_original_price | 游戏账号包月原价 |
| account_discount_price | 游戏账号包月折扣价 |
| host_original_price | Xbox主机包月原价 |
| host_discount_price | Xbox主机包月折扣价 |
| full_original_price | 全功能包月原价 |
| full_discount_price | 全功能包月折扣价 |
| points_original_price | 点数原价（per account） |
| points_discount_price | 点数折扣价（per account） |
| points_threshold | 升级到此等级需要的累计消费金额 |

#### 1.4.3 VIP等级与消费金额对应

| VIP等级 | 累计消费金额门槛 | 包月折扣 | 点数折扣 |
|---------|----------------|-------------|---------|
| VIP0 | 0 | 无折扣 | 无折扣 |
| VIP1 | 1000 | 9.5折 | 9.5折 |
| VIP2 | 5000 | 9.0折 | 9.0折 |
| VIP3 | 10000 | 8.5折 | 8.5折 |
| VIP4 | 20000 | 8.0折 | 8.0折 |
| VIP5 | 50000+ | 7.5折 | 7.5折 |

### 1.5 业务流程变化

#### 1.5.1 激活码生成（改为一次一个）

**旧流程**：
- 批量生成多个激活码
- 选择批次、时长、数量

**新流程**：
- 一次只生成一个激活码
- 选择订阅类型（流媒体账号/游戏账号/Xbox主机/全功能/点数）
- 根据类型展示不同的数据源：
  - 流媒体账号包月 → 展示商户的流媒体账号列表（多选）
  - 游戏账号包月 → 展示商户的游戏账号列表（多选）
  - Xbox主机包月 → 展示商户的Xbox主机列表（多选）
  - 全功能包月 → 不展示数据源
  - 点数充值 → 不展示数据源，展示充值点数数量

- 根据商户当前VIP等级自动带入**原价**和**折扣价**（不可编辑）
- 生成激活码

#### 1.5.2 激活码激活

**旧流程**：
- 输入激活码
- 直接激活

**新流程**：
- 输入激活码
- 查询激活码信息（类型、价格、绑定资源等）
- 如果是包月类型：
  - 获取当前商户所有已激活的订阅
  - 计算激活时间 = 最近到期时间 + 1天（或当前时间如果没有有效订阅）
  - 计算到期时间 = 激活时间 + 30天
  - 校验时间不能与现有订阅重合
  - 激活并记录

#### 1.5.3 订阅管理展示

新增"包月"标签页，展示：
- 订阅类型
- 绑定资源列表
- 开始时间
- 到期时间
- 状态（有效/已过期）

## 四、启动自动化校验流程

### 4.1 校验流程图

```
启动自动化请求
    ↓
获取当前商户所有有效订阅（当前时间在start_time和end_time之间）
    ↓
是否存在有效订阅？
    ↓ 是 ↓ 否
    ↓     ↓
获取订阅类型   提示：无有效订阅，请先购买订阅
    ↓
根据订阅类型进行资源校验
    ↓
校验通过 → 继续执行自动化
校验失败 → 返回具体错误信息
```

### 4.2 各类型详细校验逻辑

#### 类型1：window_account（流媒体账号包月）

```
1. 检查选择的流媒体账号是否在订阅的bound_resource_ids中
   - 不在 → 提示："选择的流媒体账号不在订阅范围内"
   - 在 → 继续

2. 游戏账号来源 = 该流媒体账号下的所有游戏账号

3. Xbox主机 = 商户下所有在线Xbox主机（可不选）
```

#### 类型2：account（游戏账号包月）

```
1. 游戏账号来源 = 订阅bound_resource_ids中的游戏账号
   - 过滤掉未绑定在任何流媒体账号下的游戏账号
   - 如果过滤后为空 → 提示："订阅的游戏账号都未绑定流媒体账号，无法启动自动化"

2. Xbox主机 = 商户下所有在线Xbox主机（可不选）
```

#### 类型3：host（Xbox主机包月）

```
1. Xbox主机来源 = 订阅bound_resource_ids中的Xbox主机（必选）
   - 如果为空 → 提示："订阅的Xbox主机不存在或已下线"
   - 如果有多个，只能选一个

2. 选择Xbox主机后，获取绑定在该Xbox主机上的流媒体账号

3. 游戏账号来源 = 该流媒体账号下的所有游戏账号
   - 如果为空 → 提示："该Xbox主机未绑定流媒体账号或流媒体账号下无游戏账号"
```

#### 类型4：full（全功能包月）

```
1. 不校验流媒体账号

2. 游戏账号来源 = 商户下所有游戏账号（默认全选）

3. Xbox主机 = 商户下所有在线Xbox主机（可不选）
```

#### 类型5：points（点数充值）

```
1. 不校验流媒体账号

2. 游戏账号来源 = 绑定流媒体账号下的所有游戏账号
   - 如果为空 → 提示："没有可用的游戏账号"

3. Xbox主机 = 商户下所有在线Xbox主机（可不选）

4. 点数扣除 = 游戏账号数量 × 单价（根据VIP等级）
```

## 五、点数扣除详细说明

### 5.1 点数扣除计算

```
应扣点数 = 游戏账号数量 × 单价

单价来源：
- 根据商户当前VIP等级查询merchant_group
- 如果是折扣价：使用points_discount_price
- 如果是原价：使用points_original_price
```

### 5.2 事务控制

**问题**：多个流媒体账号同时使用，可能存在并发扣点

**解决方案**：
1. 使用乐观锁（version字段）更新merchant_balance
2. 扣点前查询余额，扣点时校验余额是否变化
3. 如果变化，回滚并重试（最多3次）

```java
@Transactional(rollbackFor = Exception.class)
public boolean deductPointsWithRetry(String merchantId, int points, String userId, String reason) {
    for (int i = 0; i < 3; i++) {
        MerchantBalance balance = merchantBalanceMapper.selectByMerchantId(merchantId);
        if (balance.getBalance() < points) {
            return false;
        }

        int newBalance = balance.getBalance() - points;
        int rows = merchantBalanceMapper.deductBalanceWithVersion(
            merchantId, newBalance, balance.getVersion(), points, userId, reason
        );

        if (rows > 0) {
            return true;
        }
        log.warn("扣点失败，版本冲突，重试第{}次", i + 1);
    }
    return false;
}
```

## 六、VIP等级计算

### 6.1 消费金额记录

每当商户消费（购买激活码）时，记录消费金额：

```sql
-- 记录消费
INSERT INTO point_transaction (merchant_id, type, amount, balance_after, reason, created_time)
VALUES ('merchant_xxx', 'consume', 30000, 70000, '购买包月', NOW());

-- 更新商户累计消费
UPDATE merchant SET total_consumed = total_consumed + 30000 WHERE id = 'merchant_xxx';

-- 重新计算VIP等级
UPDATE merchant m
SET m.vip_level = (
    SELECT COALESCE(MAX(g.vip_level), 0)
    FROM merchant_group g
    WHERE m.total_consumed >= g.points_threshold
)
WHERE m.id = 'merchant_xxx';
```

### 6.2 VIP等级查询

```java
public int calculateVipLevel(int totalConsumed) {
    LambdaQueryWrapper<MerchantGroup> wrapper = new LambdaQueryWrapper<>();
    wrapper.le(MerchantGroup::getPointsThreshold, totalConsumed)
           .eq(MerchantGroup::getStatus, "active")
           .orderByDesc(MerchantGroup::getVipLevel)
           .last("LIMIT 1");
    MerchantGroup group = merchantGroupMapper.selectOne(wrapper);
    return group != null ? group.getVipLevel() : 0;
}
```
