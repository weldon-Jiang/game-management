#### ⚠️ 设计不合理之处

**问题2: AES密钥处理方式不一致**

**后端** ([AesUtil.java](d:\auto-xbox\team-management\bend-platform\src\main\java\com\bend\platform\util\AesUtil.java) 第24-34行):
```java
private SecretKeySpec getKeySpec() {
    byte[] keyBytes = secret.getBytes(StandardCharsets.UTF_8);
    int keyLen = keyBytes.length;
    if (keyLen < 16) {
        byte[] padded = new byte[16];
        System.arraycopy(keyBytes, 0, padded, 0, keyLen);
        keyBytes = padded;
        keyLen = 16;
    }
    return new SecretKeySpec(keyBytes, "AES");
}
```

**Agent端** ([crypto_util.py](d:\auto-xbox\team-management\bend-agent\src\agent\utils\crypto_util.py) 第34-55行):
```python
def get_aes_key() -> bytes:
    secret = config.get('aes.secret')
    if not secret:
        import os
        secret = os.environ.get('AES_SECRET', 'default_aes_secret_key!')

    key = secret.encode('utf-8')

    if len(key) < 16:
        key = key + b'\x00' * (16 - len(key))

    return key
```

**问题**: 后端密钥长度小于16字节时**不截断**，只补0；而Agent端密钥处理是**不截断**的。文档注释说"密钥长度 >= 16 字节时：直接使用（不截断）"，但后端的处理是**截断到16/24/32字节**。

**问题3: Redis未启用时的降级处理存在安全隐患**

[CredentialTokenService.java](d:\auto-xbox\team-management\bend-platform\src\main\java\com\bend\platform\service\CredentialTokenService.java) 第26-30行:
```java
public String generateToken(String credentialKey, String encryptedValue) {
    if (!isRedisAvailable()) {
        log.warn("Redis未启用，无法生成凭证令牌 - key: {}", credentialKey);
        return "DISABLED:" + UUID.randomUUID().toString();  // ⚠️ 安全隐患
```

当Redis未启用时，返回 `"DISABLED:" + UUID`，Agent端会将其当作无效token处理，导致任务失败，但更严重的是**加密的密码被直接传递给Agent**（在taskParams中）。

## 三、加密解密一致性检查

### 3.1 后端加密方式

**算法**: AES/ECB/NoPadding
**填充方式**: ZeroPadding (数据末尾补0x00直到16字节倍数)
**密钥处理**: 密钥不足16字节用0补足，超过16字节截断

### 3.2 Agent解密方式

**算法**: AES/ECB/NoPadding
**填充方式**: ZeroPadding (使用 `rstrip(b'\x00')` 移除)
**密钥处理**: 密钥不足16字节用0补足，超过16字节不截断（根据文档）

### 3.3 一致性分析

#### ✅ 一致的部分:
1. 算法都是AES/ECB/NoPadding
2. 填充方式都是ZeroPadding
3. 密钥不足16字节时都用0补足

#### ⚠️ 不一致的部分:
1. **密钥长度超过16字节时的处理**:
   - 后端：超过16字节会截断（`new SecretKeySpec(keyBytes, "AES")` 只会使用前16/24/32字节）
   - Agent：文档说"超过16字节则截断"，但代码注释说"密钥长度 >= 16 字节时：直接使用（不截断）"

2. **Agent端存在默认密钥** ([crypto_util.py](d:\auto-xbox\team-management\bend-agent\src\agent\utils\crypto_util.py) 第47-48行):
```python
if not secret:
    import os
    secret = os.environ.get('AES_SECRET', 'default_aes_secret_key!')
```
如果配置文件没有设置AES secret，会使用 `'default_aes_secret_key!'`，这与后端的 `aes.secret` 配置可能不一致。

### 3.4 AES密钥配置

**Agent端** ([agent.yaml](d:\auto-xbox\team-management\bend-agent\configs\agent.yaml) 第11-12行):
```yaml
aes:
  secret: "0sedzo/xcWVHjwKvJAk4lg=="
```

**后端**: 从环境变量 `AES_SECRET` 读取，默认值为空字符串

⚠️ **警告**: Agent配置中的AES密钥是Base64编码格式 `"0sedzo/xcWVHjwKvJAk4lg=="`，而后端直接使用环境变量的字符串值作为密钥。如果后端的 `AES_SECRET` 环境变量也是这个Base64字符串，则密钥处理会有问题（Base64解码后才能用于AES）。

## 四、优化建议

### 4.1 高优先级修复

#### 1. 修复Agent端token交换逻辑
**问题**: Agent端没有实现token交换
**修复位置**: [stream_control_task.py](d:\auto-xbox\team-management\bend-agent\src\agent\task\stream_control_task.py) 第321-348行

**建议实现**:
```python
def _decrypt_password(self, encrypted_password: str) -> Optional[str]:
    if not encrypted_password:
        return None

    try:
        # 如果是 token 格式，需要先换取实际密码
        if encrypted_password.startswith('token:'):
            token = encrypted_password[6:]  # 去掉 'token:' 前缀
            encrypted_password = await self._api_client.exchange_credential_token(token)
            if not encrypted_password:
                self.logger.error("Token交换失败")
                return None

        # 直接解密
        return decrypt_password(encrypted_password)

    except Exception as e:
        self.logger.error(f"密码解密失败: {e}")
        return None
```
