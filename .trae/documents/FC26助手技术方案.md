# FC26助手 - 技术方案与开发任务清单

## 一、项目概述

**项目背景**：服务足球游戏工作室的自动化工具，支持FC26（PS/Xbox/PC平台）的多窗口串流控制与灵活收费。

**核心挑战**：
1. 显卡解码多开不卡顿
2. 灵活的多平台收费系统

---

## 二、底层串流架构设计

### 2.1 硬解码方案对比

| 方案 | 适用平台 | 多开性能 | 配置复杂度 | 推荐度 |
|------|---------|---------|-----------|--------|
| **NVENC** | NVIDIA显卡 | ★★★★★ | 中 | ⭐⭐⭐⭐⭐ |
| **AMD AMF** | AMD显卡 | ★★★★☆ | 中 | ⭐⭐⭐⭐ |
| **DXVA2** | Intel/混合 | ★★★☆☆ | 高 | ⭐⭐⭐ |
| **QSV** | Intel核显 | ★★★☆☆ | 低 | ⭐⭐ |

**推荐：NVENC + Moonlight框架**
- 单卡支持20+并发串流（H.265编码）
- 低延迟（<50ms）
- 开源方案成熟

### 2.2 架构分层

```
┌─────────────────────────────────────────────────────┐
│                  B端管理后台（Web）                    │
│  - 商户管理 / 用户管理 / 订阅管理 / 收费配置             │
└─────────────────────┬───────────────────────────────┘
                      │ HTTP/WebSocket
┌─────────────────────▼───────────────────────────────┐
│               Agent集群控制层（Node.js）               │
│  - 任务调度 / 心跳检测 / 资源分配 / 负载均衡            │
└─────────────────────┬───────────────────────────────┘
                      │ gRPC/ZeroMQ
┌─────────────────────▼───────────────────────────────┐
│              宿主机串流引擎（Python/C++）               │
│  - Moonlight/AndroidController/Chiaki集成             │
│  - 多窗口管理 / 输入录制 / 反作弊规避                   │
└─────────────────────┬───────────────────────────────┘
                      │ 本地API
┌─────────────────────▼───────────────────────────────┐
│               游戏实例层（Docker/VM隔离）              │
│  - FC26 Xbox版 / FC26 PS版 / FC26 PC版               │
│  - 独立显卡解码 / 独立声卡 / 独立手柄映射               │
└─────────────────────────────────────────────────────┘
```

## 三、平台适配方案

### 3.1 Xbox平台

**硬件识别**：
```python
# 使用Xbox API识别主机型号
XboxAPI.get_console_type(console_ip)  # 返回 "XboxOne" | "XboxSeriesX" | "XboxSeriesS"
```

**多版本支持**：
| 机器类型 | 可运行版本 | 串流配置 |
|---------|-----------|---------|
| Xbox One | FC26 Xbox One版 | 1080p@30fps |
| Xbox Series X | FC26 XB1版 + XBSX版 | 4K@60fps |
| Xbox Series S | FC26 XB1版 + XSSS版 | 1440p@60fps |

**串流方案**：Sunshine + Moonlight
- Sunshine：开源PC串流服务端
- 支持HDR、ALLM、自动码率

### 3.2 PS平台

**Chiaki vs Chiaki-NG 对比**：
| 特性 | Chiaki | Chiaki-NG |
|------|--------|-----------|
| 维护状态 | 低（已停止更新） | 高（活跃开发） |
| PS5支持 | 基础 | 完整 |
| 码率控制 | 固定 | 自适应 |
| **推荐** | ❌ | ✅ |

**集成方案**：
```python
# 使用Chiaki-NG Python绑定
from chiaki_py import ChiakiSession

session = ChiakiSession(
    host=ps_ip,
    version=SessionVersion.PS5,  # 或 PS4
    video_codec=VideoCodec.H265
)
```

### 3.3 窗口隔离方案

**Docker容器化**：
```yaml
# docker-compose.yml
services:
  fc26_xbox_1:
    image: fc26-stream:latest
    runtime: nvidia
    environment:
      CONSOLE_TYPE: XboxSeriesX
      GAME_VERSION: FC26_XBSX
      DISPLAY: :0
    devices:
      - /dev/input/js0
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: 1
              capabilities: [gpu]
```

## 四、收费系统设计

### 4.1 灵活计费模式

| 模式 | 适用场景 | 示例 |
|------|---------|------|
|按时长 | 散客 | 5元/小时 |
| 按场次 | 联赛玩家 | 15元/场（90分钟） |
| 包周/月 | 工作室 | 199元/周 |
| 包窗口数 | 多开玩家 | 299元/月/5窗口 |
| 定制套餐 | 大客户 | 协商 |

### 4.2 数据模型

```
Merchant (商户)
├── id, name, isSystem
├── subscription_type: enum[HOURLY, SESSION, MONTHLY, CUSTOM]
├── rate_config: JSON  # {"hourly": 5, "session": 15}
└── max_windows: int

User (用户)
├── merchant_id
├── balance: decimal
├── role: enum[owner, admin, operator]
└── vip_level: int

Subscription (订阅)
├── user_id
├── merchant_id
├── start_time, end_time
├── windows_allocated: int
└── status: enum[active, expired, suspended]

UsageRecord (使用记录)
├── user_id
├── window_id
├── start_time, end_time
├── platform: enum[xbox, ps, pc]
└── fee_calculated: decimal
```

### 4.3 收费流程

```
用户请求开窗口
    ↓
检查订阅状态（未过期 + 窗口数未超限）
    ↓
检查余额（余额不足则提示充值）
    ↓
创建游戏实例 + 开始计时
    ↓
用户关闭窗口 / 超时
    ↓
计算费用 = 时长 × 单价
    ↓
扣减余额 + 记录使用明细
```

## 六、技术选型总结

| 模块 | 技术栈 | 说明 |
|------|--------|------|
| 串流引擎 | Python + Moonlight + Chiaki-NG | 跨平台支持 |
| 硬解码 | NVENC (NVIDIA) | 多开性能最优 |
| Agent控制 | Node.js + gRPC | 高并发、低延迟 |
| 后端服务 | Java Spring Boot | 商户管理、计费逻辑 |
| 数据库 | MySQL + Redis | 商户数据 + 缓存 |
| 前端 | Vue3 + Element Plus | B端管理后台 |
| 容器化 | Docker + NVIDIA Container Toolkit | 窗口隔离 |

## 七、风险评估

| 风险 | 影响 | 应对 |
|------|------|------|
| PS5串流延迟高 | 用户体验 | 优化码率+帧预测 |
| 多开显卡过热 | 稳定性 | 液冷/强散热方案 |
| 反作弊检测 | 账号安全 | 随机化操作间隔 |
| 各平台版本差异 | 兼容性 | 抽象适配层统一接口 |
